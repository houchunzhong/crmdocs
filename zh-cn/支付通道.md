# **支付通道**  

## **一、模块概览**

“支付通道”模块是 MIM CRM 的**资金流转控制中心**，负责定义和维护所有可用的资金出入路径。它不仅决定了用户如何向账户存入资金或提取利润，也直接影响用户体验、交易效率与反洗钱（AML）风险控制。通过本模块，管理员可以：

- ✅ 添加或编辑支付方式（如 BankTransfer、Skrill、Crypto）  
- ✅ 设置支持的币种、手续费率、单笔限额与到账时间  
- ✅ 配置支付方向（代收/代付）、关联钱包与平台支持范围  
- ✅ 优化支付排序以提升用户选择优先级  

> 🔐 仅限**管理员**或**财务主管**访问。所有更改均需保存后生效，并记录于系统日志中以供审计。

## **三、核心功能详解**

### **1. 支付通道列表视图**

#### **功能描述**  
以表格形式展示系统中已配置的所有支付通道，支持快速浏览与操作。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **ID** | 唯一标识符（如 `5`） |
| **支付显示名称** | 用户端可见的支付方式名称（如 “BankTransfer - Offline”） |
| **排序** | 决定在前端界面中的显示顺序（数值越小，优先级越高） |
| **状态** | 当前启用状态：  
  - 🟩 **开启**：正常可用  
  - 🔴 **关闭**：暂停服务 |
| **支付方向** | 资金流向：  
  - **代收**：用户向平台充值（Deposit）  
  - **代付**：平台向用户付款（Withdrawal） |
| **支持货币** | 该通道支持的币种（如 USD、GBP、EUR） |
| **支持平台** | 可用的交易系统（如 MT5、BO、AQX） |
| **钱包** | 关联的钱包类型（如 MT5&AQX钱包、BO钱包） |
| **手续费率** | 手续费百分比（如 1%） |
| **用户单笔最小 (USD)** | 单次交易最低金额（单位：USD） |
| **用户单笔最大 (USD)** | 单次交易最高金额（单位：USD） |
| **操作** | 包含“查看”和“参数”两个按钮：  
  - **查看**：进入详情页  
  - **参数**：进入高级参数设置 |

> 💡 示例：`Skrill - Paymentgateway` 是一个“代付”通道，支持 USD 和 GBP，适用于 MT5 平台，手续费为 1%，单笔最多 50,000 USD。

---

### **2. 支付通道详情页**

#### **功能描述**  
点击“查看”后进入详情页面，展示该通道的完整配置信息。

#### **配置项说明**
| 字段 | 说明 |
|------|------|
| **Logo** | 显示在用户端的图标（如 Skrill 的紫色标志） |
| **支付显示名称** | 用户看到的名称（如 “Crypto - Payment”） |
| **排序** | 前端显示顺序（数字越小，越靠前） |
| **状态** | 启用或禁用 |
| **支付方向** | 代收（Deposit）或代付（Withdrawal） |
| **支持货币** | 支持的币种列表（如 USDT、USDC） |
| **支持平台** | 适用的交易系统（如 MT5、BO、AQX） |
| **钱包** | 关联的钱包（如 MT5&AQX钱包） |
| **通道类型** | 分类：线上 / 线下（T/T） |
| **付款方式** | 具体支付手段（如 “银行转账 (T/T)”） |
| **预计到账时间** | 资金到达用户的预估时长（如 “1小时”、“2天”） |
| **手续费率** | 手续费比例（如 2%） |
| **手续费承担方** | 谁支付手续费：  
  - 用户支付  
  - 平台支付 |
| **通道单笔最小 (USD)** | 该通道允许的最小交易额（系统级限制） |
| **通道单笔最大 (USD)** | 该通道允许的最大交易额（系统级限制） |
| **用户单笔最小 (USD)** | 用户端可输入的最小金额（通常 ≤ 通道最小） |
| **用户单笔最大 (USD)** | 用户端可输入的最大金额（通常 ≤ 通道最大） |
| **支付通道备注（仅管理员可见）** | 内部注释，不对外展示（如 “test”、“3”） |

> ⚠️ 注意：  
> - “用户单笔”限制用于前端输入校验，“通道单笔”为后端实际处理上限  
> - 若“手续费承担方”为“平台”，则用户无需支付费用

---

### **3. 参数设置页面**

#### **功能描述**  
点击“参数”按钮进入高级配置页面，用于自定义支付过程中的字段与值。

#### **典型应用场景**
- **银行转账（T/T）**：配置收款账户信息（如银行名称、账号、SWIFT/BIC）  
- **加密货币支付**：设置钱包地址、网络类型（ERC-20、TRC-20）等  

#### **字段示例（以 Crypto - Payment 为例）**
| 排序 | 字段 | 值 |
|------|------|-----|
| 1 | 银行名称 | Bank of America |
| 2 | 账户号码 | 1234567890 |
| 3 | SWIFT/BIC | BOFAUS3N |
| 4 | 收款人姓名 | TradeWise Limited |

> 💡 操作：可新增、删除或修改字段，支持多条记录。完成后点击“确认”保存。

## **四、操作流程示例**

### **场景：添加新的支付通道 —— “Crypto - Tron”**

1. 进入 **支付通道** → 点击 **新增**  
2. 填写基本信息：
   - 支付显示名称：`Crypto - Tron`  
   - 排序：`7`  
   - 状态：`开启`  
   - 支付方向：`代收`  
   - 支持货币：`USDC`, `USDT`  
   - 支持平台：`BO`  
   - 钱包：`MT5&AQX钱包`  
   - 手续费率：`0%`  
   - 用户单笔最小：`500`  
   - 用户单笔最大：`10,000`  
3. 点击 **保存**  
4. 在 **参数** 页面配置 TRON 网络的接收地址：
   - 字段：`钱包地址` → 值：`TJ...abc123`  
   - 字段：`网络类型` → 值：`TRC-20`  
5. 点击 **确认** 完成设置  

> ✅ 成功后，用户可在“充值”页面选择“Crypto - Tron”进行 USDC/USDT 充值。

---

### **场景：调整 Skrill 提现手续费**

1. 找到 `Skrill - Paymentgateway` → 点击 **查看**  
2. 进入详情页 → 修改“手续费率”为 `1.5%`  
3. 点击 **编辑** → 保存  
4. 系统自动更新，后续所有 Skrill 提现将按新费率执行

## **五、最佳实践建议**

| 场景 | 推荐做法 |
|------|----------|
| **多币种支持** | 优先配置主流法币（USD、EUR、GBP）和稳定币（USDT、USDC） |
| **手续费策略** | 对高价值用户可设置“平台承担手续费”以增强体验 |
| **到账时间** | 明确标注预期到账时间，避免用户投诉 |
| **安全性** | 所有线下支付必须绑定实名账户，防止洗钱风险 |
| **合规要求** | 确保所有支付方式符合当地金融监管规定（如不支持非法虚拟资产） |

## **六、常见问题解答（FAQ）**

### **Q1：为什么某个支付通道无法使用？**  
- ❌ 可能原因：  
  - 状态为“关闭”  
  - 不支持当前用户所在地区  
  - 未绑定有效钱包或API密钥  

> ✅ 解决方案：检查状态、区域限制与钱包配置。

---

### **Q2：“代收”和“代付”有什么区别？**  
- ✅ **代收**：用户向平台充值（Deposit），资金流入系统  
- ✅ **代付**：平台向用户付款（Withdrawal），资金流出系统  

> 💡 示例：Skrill 是“代付”通道，用于用户提现；BankTransfer 是“代收”通道，用于用户充值。

---

### **Q3：能否同时支持多个支付方式？**  
- ✅ 是的。系统支持多种通道并存，用户可自由选择最合适的支付方式。

---

### **Q4：如何设置加密货币钱包地址？**  
- ✅ 方法：  
  1. 进入对应支付通道 → 点击 **参数**  
  2. 新增字段：`钱包地址` → 输入真实地址  
  3. 可选：添加 `网络类型`（如 ERC-20、TRC-20）  
  4. 保存即可

---

### **Q5：支付通道是否影响佣金结算？**  
- ✅ 是的。如果某通道用于“代收”，其手续费可能影响最终到账金额，进而影响代理佣金计算。建议在配置时考虑成本因素。

## **七、总结**

“支付通道”模块是 MIM CRM 实现全球化、多元化资金流动的核心组件。通过精细化配置每种支付方式的属性与规则，系统能够满足不同用户群体的需求，兼顾效率、安全与合规性。无论是接入传统银行转账、第三方支付网关，还是支持加密货币，本模块都提供了强大的灵活性与可控性。

1. 进入 **支付通道** → 2. 点击 **新增** → 3. 填写基础信息 → 4. 设置币种、费率与限额 → 5. 点击 **保存** → 6. 进入 **参数** 配置具体字段 → 7. 点击 **确认**

> 📌 所有变更均被记录在系统日志中，支持审计与回溯，确保财务管理的透明性与合规性。
