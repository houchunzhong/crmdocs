# **管理员模块**  

## **一、模块概览**

“管理员”模块是 MIM CRM 的**权限中枢**，包含两大核心功能：

- ✅ **管理员列表（Administrator List）**：查看、添加和编辑系统管理员账户  
- ✅ **系统日志（System Logs）**：追踪所有管理员的操作行为，支持按时间、IP、用户等维度查询  

> 🔐 仅限 **Master 级别管理员** 操作，所有变更均记录在审计日志中，确保可追溯性。

## **三、核心功能详解**

### **1. 管理员列表（Administrator List）**

#### **功能描述**  
展示系统中所有管理员账户的基本信息，支持新增、编辑和状态控制。每个管理员拥有唯一 ID 和角色权限，决定其可访问的功能范围。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **ID** | 管理员唯一编号（自增） |
| **姓名** | 用户显示名称（如 “Default Admin”） |
| **手机** | 联系电话（用于登录验证） |
| **邮箱** | 登录账号（必须唯一） |
| **角色权限** | 分配的角色（如 Master, Manager, Operator） |
| **创建时间** | 账户创建日期（格式：DD/MM/YYYY） |
| **最后登录时间** | 最近一次成功登录的时间 |
| **最后登录IP** | 最近登录的 IP 地址（用于安全分析） |
| **状态** | `启用` / `禁用`（通过开关控制） |
| **操作** | **编辑**（修改信息） |

#### **操作流程**
1. 点击 **“添加管理员”** 按钮。
2. 填写新管理员信息：
   - 姓名
   - 手机号（需验证）
   - 邮箱（需唯一）
   - 角色权限（从下拉菜单选择）
3. 设置初始密码或发送重置链接。
4. 点击 **“保存”** 完成创建。
5. 新管理员将出现在列表中，并可在“系统日志”中查看其操作。

> 💡 示例：若需创建一名财务专员，可为其分配 “Operator” 角色，仅允许访问资金与支付模块。

---

### **2. 系统日志（System Logs）**

#### **功能描述**  
记录所有管理员执行的关键操作，包括登录、数据修改、配置变更等。支持多维度筛选，便于快速定位异常行为。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **名称** | 管理员姓名 |
| **邮箱** | 登录邮箱 |
| **手机** | 关联手机号 |
| **操作名** | 操作类型（如 LOGIN, UPDATE, DELETE） |
| **对象类型** | 操作目标（如 ADMINISTRATOR, CURRENCY, PAYMENT_CHANNEL） |
| **关联对象** | 具体操作对象（如 “Administrator” 或 “USDT”） |
| **操作详情** | 简要描述（如 “Login”, “Updated currency rate”） |
| **创建时间** | 操作发生时间（格式：日/月/年） |
| **IP 地址** | 操作来源 IP（可用于地理定位） |

#### **搜索与过滤功能**
- **开始日期 ～ 结束日期**：按时间范围筛选
- **名称/手机号查询**：输入关键词查找特定管理员
- **IP 地址查询**：输入 IP 追踪可疑登录

#### **操作流程**
1. 进入 **“系统日志”** 页面。
2. 使用顶部筛选器设置查询条件。
3. 点击 **“搜索”** 查找相关记录。
4. 查看操作详情，确认是否正常。

> 📌 注意：所有日志保留至少 **180 天**，符合金融行业合规要求。

## **四、角色权限体系（Role-Based Access Control）**

| 角色 | 权限说明 |
|------|----------|
| **Master** | 全局最高权限，可管理所有模块、创建其他管理员、修改系统配置 |
| **Manager** | 可管理用户、资金、交易账户、代理等核心业务模块 |
| **Operator** | 有限权限，通常用于财务、客服岗位，仅能查看或执行指定任务 |

> ⚠️ 建议：遵循最小权限原则，避免过度授权。

## **五、最佳实践建议**

| 场景 | 建议 |
|------|------|
| **首次部署** | 创建一个 Master 账户作为主控，其余人员使用子账户 |
| **员工离职** | 立即禁用其账户，防止未授权访问 |
| **敏感操作** | 所有重要变更需双人复核，避免误操作 |
| **定期审计** | 每月检查系统日志，识别异常登录行为 |

## **六、常见问题解答（FAQ）**

### **Q1：如何添加新的管理员？**  
- 在“管理员列表”页面点击 **“添加管理员”**，填写信息并分配角色即可。

### **Q2：为什么无法看到某个操作？**  
- 请确认筛选条件是否正确，或检查是否有权限查看该模块的日志。

### **Q3：系统日志可以导出吗？**  
- ✅ 可以。点击右上角 **“导出 CSV”** 按钮，下载加密压缩包（需输入交易密码解压）。

### **Q4：管理员账户可以删除吗？**  
- ❌ 不可以。只能禁用。删除会导致历史日志丢失，影响审计。

## **七、总结**

MIM CRM 的“管理员”模块为企业提供了强大的权限控制与操作审计能力。通过清晰的权限分级和完整的日志记录，系统不仅保障了后台安全，也为合规审查提供了坚实支持。

✅ **推荐操作顺序**：  
1. 创建 Master 账户 → 2. 添加子管理员 → 3. 分配角色 → 4. 定期检查日志 → 5. 必要时禁用账户

> 📌 所有操作均需遵守公司内部安全政策，确保系统稳定运行。
