
## **一、模块概览**

“个人中心”作为用户登录后的默认门户之一，集成了身份、财务与安全三大维度的自助管理功能，确保用户在合规前提下高效完成日常操作。

### **核心子模块**
- ✅ **我的资料（My Profile）**：查看与更新个人身份信息  
- ✅ **资金账户登记（Funding Accounts）**：绑定并管理入金/出金账户  
- ✅ **安全设置（Security Settings）**：配置登录验证与密码策略  
- ✅ **操作日志（Activity Log）**：追踪关键操作记录  

> 🔐 所有敏感操作（如修改身份证信息、删除提款账户）均需二次验证，并同步触发系统通知。

## **三、核心功能详解**

### **1. 我的资料（My Profile）**

#### **功能描述**  
展示用户已通过 KYC 审核的核心身份信息，并允许在合规范围内更新非关键字段（如邮箱、手机号）。

#### **字段说明**
| 字段 | 是否可编辑 | 说明 |
|------|-----------|------|
| 姓名 | ❌ | 与身份证件一致，不可修改 |
| 国籍 / 居住国家 | ❌ | 由 KYC 审核锁定 |
| 手机号 | ✅（需验证） | 修改后需短信验证码确认 |
| 邮箱 | ✅（需验证） | 修改后需点击邮件链接激活 |
| 用户等级 | ❌ | 系统自动计算（基于交易量或 IB 下级数） |
| KYC 状态 | ❌ | 显示 `已验证` + 最后审核时间 |
| 资料来源 | ❌ | 标注为 `用户提交` 或 `系统导入` |

#### **操作流程**
1. 点击 **“编辑”** 按钮（仅对可编辑字段生效）。
2. 输入新手机号/邮箱。
3. 系统发送验证码至原设备。
4. 用户输入验证码并保存。
5. 更新成功后，系统记录变更日志并发送通知。

> ⚠️ 注意：若用户需更改姓名或证件信息，必须重新提交完整 KYC 申请，走人工审核流程。

---

### **2. 资金账户登记（Funding Accounts）**

#### **功能描述**  
用于绑定和管理用于入金（Deposit）与出金（Withdrawal）的账户，包括银行卡、USDT 钱包、支付宝等。

#### **账户类型支持**
- 💳 **银行账户**（支持多币种：USD, EUR, GBP 等）
- 🪙 **数字钱包**（USDT-TRC20, USDT-ERC20, BTC, ETH）
- 📱 **第三方支付**（Alipay, WeChat Pay, Skrill）

#### **字段说明**
| 字段 | 说明 |
|------|------|
| 账户类型 | 自动识别（如 “USDT-TRC20”） |
| 收款人姓名 | 必须与 KYC 姓名一致 |
| 账号 / 地址 | 高亮显示（部分字符掩码保护） |
| 币种 | 如 USD, USDT |
| 国家 | 账户所属国家 |
| 状态 | `待审核` / `已验证` / `已停用` |
| 操作 | **设为默认** / **编辑** / **删除** |

#### **操作流程**
1. 点击 **“新增账户”**。
2. 选择账户类型，填写完整信息。
3. 提交后，状态变为 `待审核`。
4. 合规团队审核通过后，状态更新为 `已验证`。
5. 用户可将任一已验证账户设为 **默认出金账户**。

> 💡 建议：每个币种至少绑定一个默认账户，避免提款失败。

---

### **3. 安全设置（Security Settings）**

#### **功能描述**  
强化账户安全，防止未授权访问。

#### **子功能**
| 功能 | 说明 |
|------|------|
| **修改密码** | 需输入原密码 + 新密码（8位以上，含大小写+数字） |
| **双重验证（2FA）** | 支持 Google Authenticator 或短信验证 |
| **登录设备管理** | 查看当前活跃会话，可远程登出可疑设备 |
| **安全问题** | （可选）设置密保问题用于找回账户 |

#### **操作示例：启用 2FA**
1. 点击 **“启用 Google Authenticator”**。
2. 使用 Authenticator App 扫描二维码。
3. 输入生成的 6 位动态码。
4. 系统提示“2FA 已启用”，后续登录需输入验证码。

> 🔒 强制策略：所有 IB 和管理员账户必须开启 2FA。

---

### **4. 操作日志（Activity Log）**

#### **功能描述**  
提供透明、可追溯的操作审计记录，满足用户自查与合规需求。

#### **记录内容**
| 字段 | 示例 |
|------|------|
| 操作类型 | `登录`、`修改邮箱`、`新增提款账户`、`发起提款` |
| 操作时间 | `2026-01-27 14:30:22` |
| IP 地址 | `114.242.xxx.xxx (Beijing, CN)` |
| 设备 | `Chrome on Windows` |
| 状态 | `成功` / `失败（密码错误）` |

#### **使用场景**
- 怀疑账户被盗？→ 查看异常登录 IP  
- 忘记是否提交了提款？→ 搜索“Withdrawal”操作  
- 配合客服调查？→ 导出日志 CSV 文件  

> 📌 保留周期：操作日志保存 **180 天**，符合金融行业标准。

## **四、设计风格与用户体验原则**

### **1. 一致性（Consistency）**
- 沿用 MIM CRM 全局设计语言：
  - 主色调：深蓝 (#0A2540) + 科技蓝 (#007BFF)
  - 图标：线性简约风格（如 👤、💳、🔒）
  - 按钮：圆角矩形，主操作高亮

### **2. 安全优先（Security-First）**
- 敏感信息自动脱敏（如银行卡号显示为 `****1234`）
- 关键操作需二次确认（弹窗 + 验证码）
- 所有页面强制 HTTPS + 会话超时（15 分钟无操作自动登出）

### **3. 自助化（Self-Service）**
- 90% 的常规操作无需联系客服
- 每个功能页嵌入 **“?” 帮助图标**，链接至对应 FAQ
- 错误提示友好（如“手机号格式错误，请输入+86开头的11位号码”）

### **4. 多语言支持**
- 界面语言随用户偏好自动切换（中/英/日/印尼语等）
- 日期格式、数字分隔符本地化（如 `1,000.00` vs `1.000,00`）

## **六、常见问题解答（FAQ）**

### **Q1：为什么我不能修改姓名？**  
- 姓名由 KYC 审核锁定，如需变更，必须重新提交身份证明文件并走人工审核流程。

### **Q2：删除提款账户后还能找回吗？**  
- ❌ 不能。删除即永久移除，需重新提交并审核。

### **Q3：操作日志能导出吗？**  
- ✅ 可以。点击右上角 **“导出 CSV”** 按钮，下载加密压缩包（需输入交易密码解压）。

### **Q4：2FA 丢失怎么办？**  
- 联系客服，提供身份证明 + 最近交易记录，经人工核验后可重置。
