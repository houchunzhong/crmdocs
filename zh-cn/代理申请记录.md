# **代理申请记录模块使用说明书**  

## **一、模块概览**

“代理申请记录”模块是 MIM CRM 的**代理准入控制中枢**，负责接收并处理用户主动提交的“成为代理”申请。系统自动将符合条件的用户纳入待审列表，并提供完整的个人信息与KYC状态供管理员评估。通过此模块，企业可确保仅授权具备资质、信誉良好的用户担任代理，有效防范欺诈风险与合规隐患。

> 🔐 仅限**管理员**或**运营主管**访问。所有审核操作均记录于系统日志，确保流程透明、可追溯。

## **三、核心功能详解**

### **1. 代理申请列表主视图**

#### **功能描述**  
以表格形式展示所有已提交但尚未处理的代理申请，支持按条件筛选与排序，便于批量管理和优先级分配。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **用户 ID** | 系统唯一用户编号（如 `16686`） |
| **姓名** | 用户显示名（如 `Léa Petit`），附带状态图标：  
  - ✅ **绿色 ✓**：已完成基本资料填写  
  - ⚠️ **橙色 !**：信息未完整填写（如“未填写”） |
| **国籍** | 用户注册时填写的国家 |
| **居住地** | 用户当前常住地区（可与国籍不同） |
| **手机** | 联系电话，附带验证状态图标：  
  - ✅ **绿色 ✓**：已验证手机号 |
| **邮箱** | 注册邮箱，附带验证状态图标：  
  - ✅ **绿色 ✓**：已验证邮箱 |
| **用户等级** | 当前账户类型（如 `Verified Client`） |
| **注册时间** | 用户首次注册账号的时间 |
| **申请时间** | 提交代理申请的具体时间 |
| **操作** |  
  - **查看**：进入该用户的个人详情页  
  - **设为代理**：启动代理审批流程 |

#### **交互操作**
- 🔎 **搜索框**：输入 `ID`、`姓名`、`手机号` 或 `邮箱` 进行全局检索。
- 📆 **时间筛选器**：设置“提交时间从”到“至”，过滤特定时间段内的申请。
- 📊 **排序**：点击列标题可按升序/降序排列（如按“申请时间”倒序查看最新申请）。

> 💡 示例：若某用户在法国注册且已验证联系方式，则其申请可信度较高，适合优先审核。

### **2. 设为代理流程（审批操作）**

#### **功能描述**  
点击“设为代理”按钮后，系统弹出配置窗口，允许管理员为申请人指定代理角色、等级与佣金方案，完成正式授权。

#### **配置项说明**
| 字段 | 说明 |
|------|------|
| **用户类型** | 固定为 “代理”（不可更改） |
| **用户等级** | 必填项，选择该代理的初始级别：  
  - `IB`（Introducing Broker）——独立经纪人  
  - 可后续升级为更高级别（如 Master IB、VIP IB） |
| **方案类型** | 必填项，决定佣金策略来源：  
  - `跟随用户等级`：继承上级或系统默认的佣金规则  
  - `自定义方案`：手动设定专属提成比例（需提前配置） |

#### **操作步骤**
1. 在“代理申请记录”列表中找到目标用户（如 Léa Petit）
2. 点击右侧 **“设为代理”** 按钮
3. 在弹窗中：
   - 选择 **用户等级**（如 `IB`）
   - 选择 **方案类型**（如 `跟随用户等级`）
4. 点击 **“确认”** 完成审批

> ✅ 成功后，该用户将：
- 自动升级为 **IB** 角色
- 出现在 **代理列表** 中
- 获取专属 **邀请链接** 和 **邀请码**
- 开始累积下级客户并产生佣金

### **3. 用户详情页（审核参考）**

#### **功能描述**  
点击“查看”按钮可进入申请人个人资料页，全面了解其背景信息、资产状况与KYC状态，辅助做出是否批准的决策。

#### **关键信息点**
| 类别 | 内容 |
|------|------|
| **基本信息** | 姓名、国籍、居住地、注册时间等 |
| **联系方式** | 手机、邮箱（是否已验证） |
| **资金情况** | 总入金、总出金、总资产、可用余额 |
| **KYC状态** | 是否已完成身份验证（如上传护照/身份证） |
| **双因素认证** | 是否启用 2FA（增强安全性） |
| **上级用户信息** | 若已有推荐人，可查看其层级关系 |
| **用户邀请码 / 链接** | 用于推广的专属代码与URL |

> 📌 示例：Léa Petit 的资料显示她来自法国，已验证手机号和邮箱，拥有 $500 入金记录，KYC 已通过 —— 符合优质代理候选人标准。

## **四、最佳实践建议**

| 场景 | 推荐做法 |
|------|----------|
| **新申请审核** | 优先处理信息完整、已验证联系方式且有资金行为的用户 |
| **风险识别** | 对未填写姓名、无入金记录或KYC未完成的申请暂缓审批 |
| **批量处理** | 每日定时审查一次，避免积压 |
| **合规要求** | 所有代理必须满足KYC要求，否则不得升级 |
| **激励引导** | 在审批后发送欢迎邮件，介绍代理权益与佣金机制 |

## **五、常见问题解答（FAQ）**

### **Q1：为什么有些用户无法看到“设为代理”按钮？**  
- ❌ 可能原因：  
  - 用户尚未提交申请（不在本列表中）  
  - 用户已被设为代理（已在“代理列表”中）  
  - 用户存在高风险标记（如黑名单、KYC拒绝）

### **Q2：能否修改已批准代理的佣金方案？**  
- ✅ 可以。进入“代理列表” → 打开该代理详情 → 在“代理详情”页编辑“佣金方案”。

### **Q3：申请被驳回后还能再次申请吗？**  
- ✅ 可以。系统不阻止重复申请，但建议通知用户改进资料后再提交。

### **Q4：如何防止虚假代理申请？**  
- ✅ 措施包括：  
  - 强制KYC验证  
  - 要求实名注册  
  - 审核是否有真实交易行为  
  - 设置最低入金门槛（如 $100）

### **Q5：代理申请多久会过期？**  
- ❌ 当前版本无自动过期机制。未处理的申请将长期保留，建议定期清理。

“代理申请记录”模块实现了从“用户申请”到“正式授权”的标准化流程，确保代理招募过程安全、可控、可审计。通过结合用户背景、财务数据与合规状态进行综合评估，企业能够精准筛选高质量合作伙伴，构建可持续发展的代理生态体系。

1. 登录系统 → 2. 进入“代理申请记录” → 3. 搜索或筛选目标用户 → 4. 点击“查看”核实信息 → 5. 点击“设为代理” → 6. 配置等级与方案 → 7. 点击“确认”完成审批

> 📌 所有操作均有日志留存，确保每一步都清晰可查，符合监管与内部治理要求。
