# **代理列表模块使用说明书**  

## **一、模块概览**

“代理列表”模块是 MIM CRM 的**代理关系管理中枢**，通过结构化表格展示所有代理用户的综合信息，包括：

- ✅ 用户基本信息（姓名、联系方式）
- ✅ 代理等级与状态
- ✅ 佣金方案配置
- ✅ 下级客户数量与资金表现
- ✅ 当月入金/出金数据
- ✅ 总佣金与发放情况

> 🔐 仅限**管理员**或**财务主管**访问。所有变更均记录于审计日志，确保操作可追溯。

## **三、核心功能详解**

### **1. 代理列表主视图**

#### **功能描述**  
以表格形式展示所有代理用户的关键指标，支持按条件筛选与排序，便于快速评估代理绩效与业务规模。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **用户** | 代理用户名、全名及 ID（如 `Akina Akiyama - 16672`） |
| **手机&邮箱** | 联系方式，附带验证状态图标：  
  - ✅ **绿色 ✓**：已验证手机号/邮箱  
  - ❌ **红色 !**：未验证或异常 |
| **用户等级** | 当前角色等级（如 `Verified Client`, `IB`） |
| **佣金方案** | 所绑定的佣金计划名称（如 “Blank Commission Plan for initial setup”） |
| **下级用户数量** | 该代理直接推荐的客户总数（如 4 表示有 4 名下级） |
| **客户本月入金** | 其下级客户的当月累计充值金额（绿色字体显示） |
| **客户本月出金** | 其下级客户的当月累计提现金额（红色字体显示） |
| **总佣金** | 根据规则计算的累计应得佣金总额（基于入金等） |
| **已发佣金 / 未发佣金** | 已实际发放 vs 待结算的佣金金额（格式：`X.XX / Y.YY`） |
| **操作** | **查看** — 进入该代理的详情页 |

#### **交互操作**
- 🔎 **搜索框**：输入 `ID`、`姓名`、`手机号` 或 `邮箱` 即可全局检索。
- ▼ **筛选器**：选择“用户等级”过滤不同类型的代理（如只看 IB）。
- 📊 **排序**：点击列标题可按升序/降序排列（如按“客户本月入金”从高到低排序）。

> 💡 示例：若某 IB 的“客户本月入金”为 $12,000，则其贡献显著，适合重点扶持。

### **2. 代理详情页（Agent Profile）**

#### **功能描述**  
点击任一代理右侧的 **“查看”** 按钮，即可进入其个人详情页，全面掌握其账户信息、层级关系、佣金记录与结算历史。

#### **页面结构**
##### **左侧信息栏**
| 字段 | 说明 |
|------|------|
| **头像 & 姓名** | 用户头像与全名 |
| **ID & 角色** | 用户编号与类型（如 `Verified Client`） |
| **总入金 / 总出金** | 累计资金流动总额 |
| **总资产 / 可用余额** | 实时资产快照 |
| **国籍 / 居住地** | 地理位置信息 |
| **注册时间** | 账户创建时间 |
| **联系方式** | 手机号与邮箱，附带验证状态图标 |

##### **右侧内容区（Tab 切换）**
| Tab | 内容 |
|-----|------|
| **代理详情** | 编辑代理信息、佣金方案、结算方式等 |
| **用户关系树** | 查看该代理及其下级客户的完整层级结构 |
| **佣金记录** | 查看每笔交易产生的佣金明细 |
| **佣金结算记录** | 查看已审核并发放的佣金结算单 |

### **3. 关键子模块说明**

#### **① 代理详情（Agent Details）**
- **用户类型**：如 “代理”、“Verified Client”
- **用户等级**：当前权限级别
- **用户状态**：启用 / 停用
- **成为代理时间**：首次被授权为 IB 的日期
- **方案类型**：如 “跟随用户等级”（自动继承上级策略）
- **开户方案**：默认开立的交易账户模板
- **佣金方案**：当前生效的提成规则（如 “无佣金” 或 “1% 入金返佣”）
- **结算方案**：如何处理佣金支付（如 “无结算”、“手动结算”）
- **上级用户ID / 姓名**：指明其推荐人（即上级 IB）
- **用户邀请码 / 邀请链接**：用于推广的专属代码与 URL

> ✅ 示例：在 Akina Akiyama 的详情页中，可见她是由 ID:1 的用户推荐，并拥有一个“无佣金”的初始设置——表明她尚未激活正式提成计划。

#### **② 佣金记录（Commission Records）**
- 显示每一笔交易触发的佣金生成记录。
- 包括：
  - 交易单号
  - 佣金类型（如 “Deposit Commission”）
  - 代理佣金金额
  - 交易用户
  - 交易账号
  - 交易品种
  - 交易量
  - 佣金计算结果

> ⚠️ 当前为空，表示暂无交易产生佣金。

#### **③ 佣金结算记录（Commission Settlement Records）**
- 记录已完成的佣金结算流程。
- 包括：
  - 结算单号
  - 结算情况（如 “已结算”、“待审核”）
  - 审核方式（如 “系统自动”、“人工审核”）
  - 结算方案
  - 代理佣金总额
  - 结算后 CRM 余额

> 📌 注：此模块用于财务对账与支付凭证管理。

## **四、最佳实践建议**

| 场景 | 推荐做法 |
|------|----------|
| **代理招募** | 在“代理申请记录”中及时审核新申请人，避免流失优质资源 |
| **绩效评估** | 按“客户本月入金”排序，识别高价值代理进行奖励或培训 |
| **佣金配置** | 为活跃代理配置合理的佣金方案（如 1%-3%），提升积极性 |
| **风险控制** | 对长期无下级或无入金的代理进行提醒或暂停资格 |
| **财务对账** | 定期核对“佣金结算记录”，确保资金准确发放 |

## **五、常见问题解答（FAQ）**

### **Q1：如何添加新代理？**  
- 进入 **代理申请记录** → 审核通过 → 自动升级为 IB 并生成对应记录。

### **Q2：为什么某个代理的“总佣金”为 0？**  
- 可能原因：  
  - 未绑定有效佣金方案  
  - 下级客户尚未完成入金  
  - 佣金规则设为“无佣金”  

### **Q3：能否修改代理的佣金方案？**  
- ✅ 可以。在“代理详情”页点击 **“编辑”** 修改“佣金方案”。

### **Q4：代理的“下级用户数量”包含哪些人？**  
- 仅包含其**直接推荐**的客户（一级下级）。不包括二级、三级等间接用户。

### **Q5：佣金何时结算？**  
- 默认为**手动结算**，需管理员在“佣金发放记录”中发起操作。也可配置为“每日自动结算”。

## **六、总结**

“代理列表”模块为企业提供了透明、高效的代理管理体系。通过集中展示代理业绩、资金表现与佣金状态，管理员能够精准识别优秀合作伙伴，优化激励机制，保障收益分配公平合理。

✅ **推荐操作流程**：  
1. 查看“代理列表” → 2. 按“客户本月入金”排序 → 3. 点击“查看”进入详情 → 4. 审查佣金方案与结算记录 → 5. 必要时调整配置或发放佣金

> 📌 所有操作均记录于系统日志，确保过程合规、决策有据。
