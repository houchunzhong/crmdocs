# **货币与支付**

## **一、模块概览**

“货币与支付”模块是 MIM CRM 的财务中枢，涵盖两大核心子系统：

- ✅ **货币管理（Currency Management）**：定义支持的法币与数字货币，配置显示规则与换算汇率
- ✅ **支付通道管理（Payment Channel Management）**：接入并配置银行转账、第三方支付、加密钱包等入金/出金方式

> 🔐 仅限 **系统管理员** 或 **财务主管** 操作，所有变更均记录审计日志，确保可追溯性。

## **三、核心功能详解**

### **1. 货币管理（Currency Management）**

#### **功能描述**

用于添加、修改或删除系统支持的货币类型，包括法币（如 USD, PHP）和数字货币（如 USDT, BTC）。每种货币可独立设置：

- 显示格式（小数位数、千分位符号）
- 报价方式（手动 / 自动）
- 汇率（入金/出金）

#### **字段说明**

| 字段                   | 说明                                                     |
| ---------------------- | -------------------------------------------------------- |
| **货币简称**     | 唯一标识符（如 `USDT`），用于系统内部调用              |
| **货币名称**     | 用户可见名称（如 “Tether”）                            |
| **货币类型**     | `数字货市` 或 `法币`                                 |
| **显示小数位数** | 用户端展示时的小数精度（如 2 表示 $1.00）                |
| **报价方式**     | `手动`（由管理员设定）或 `自动`（对接 API 实时更新） |
| **换算方式**     | 基准货币对（如 `USDT/USD`）                            |
| **入金汇率**     | 用户存入该货币时的兑换比例（例如 1 USDT = 1.00003 USD）  |
| **出金汇率**     | 用户提取该货币时的兑换比例（例如 1 USDT = 0.99998 USD）  |
| **编辑**         | 修改当前货币配置                                         |

#### **操作流程**

1. 点击 **“新增”** 按钮。
2. 填写货币信息：
   - 选择 **货币简称** 和 **名称**
   - 设置 **货币类型**（数字货市 / 法币）
   - 配置 **显示格式**（小数位数、千分位分隔符）
   - 选择 **报价方式**（手动/自动）
   - 设置 **换算基准**（如 USDT/USD）
   - 输入 **入金汇率** 和 **出金汇率**
3. 点击 **“保存”** 完成配置。
4. 新增货币将出现在列表中，并可在后续支付通道中启用。

> 💡 示例：若需支持 USDC，则应创建新货币条目，设置其与 USD 的汇率关系。

### **2. 支付通道管理（Payment Channel Management）**

#### **功能描述**

用于配置和管理所有可用的资金出入通道，包括银行转账、第三方支付平台（如 Skrill）、加密钱包（如 TRON、ERC20）等。每个通道可独立设置：

- 支持的货币
- 手续费率
- 单笔金额限制
- 是否启用

#### **字段说明**

| 字段                              | 说明                                                        |
| --------------------------------- | ----------------------------------------------------------- |
| **ID**                      | 通道唯一编号                                                |
| **支付显示名称**            | 用户端可见名称（如 “Bank Transfer”）                      |
| **排序**                    | 列表展示顺序（数值越小越靠前）                              |
| **状态**                    | `开启` / `关闭`                                         |
| **支付方向**                | `代收`（收款）或 `代付`（付款）                         |
| **支持货币**                | 可处理的货币类型（如 BTC, USD）                             |
| **支持平台**                | 对接的交易系统（如 BO, MT5, AQX）                           |
| **钱包**                    | 关联的钱包账户（如 “MT5&AQX钱包”）                        |
| **手续费率**                | 以百分比形式表示（如 0%）                                   |
| **用户单笔最小/最大 (USD)** | 用户每次交易的金额范围                                      |
| **操作**                    | **查看**（查看详情） / **参数**（进入高级设置） |

#### **操作流程**

1. 点击 **“新增”** 按钮。
2. 填写基础信息：
   - 支付显示名称
   - 排序值
   - 状态（默认开启）
   - 支付方向（代收/代付）
   - 支持货币（可多选）
   - 支持平台
   - 钱包关联
3. 设置费用与限额：
   - 手续费率（如 0% 表示免费）
   - 用户单笔最小/最大金额（单位 USD）
4. 点击 **“保存”** 后，系统生成通道 ID 并进入 **“支付通道详情”** 页面。

---

### **3. 支付通道详情（Payment Channel Details）**

#### **功能描述**

提供支付通道的完整配置视图，支持精细化参数调整。

#### **字段说明**

| 字段                              | 说明                               |
| --------------------------------- | ---------------------------------- |
| **Logo**                    | 通道图标（如银行标志）             |
| **支付显示名称**            | 用户端名称                         |
| **排序**                    | 展示顺序                           |
| **状态**                    | 是否启用                           |
| **支付方向**                | 代收 / 代付                        |
| **支持货币**                | 具体支持的币种                     |
| **支持平台**                | 如 BO, MT5, AQX                    |
| **钱包**                    | 关联的钱包地址                     |
| **通道类型**                | 如 “线上”、“线下”              |
| **付款方式**                | 如 “银行转账 (T/T)”              |
| **预计到账时间**            | 如 “0分钟”（即时到账）           |
| **支付通道备注**            | 仅管理员可见的内部说明             |
| **手续费率**                | 以百分比表示（如 -% 表示无手续费） |
| **手续费承担方**            | “商户支付” 或 “客户支付”       |
| **通道单笔最小/最大 (USD)** | 系统级交易限额（区别于用户限额）   |
| **用户单笔最小/最大 (USD)** | 用户端可提交的金额范围             |

#### **操作流程**

1. 在支付通道列表中点击 **“查看”** 或 **“参数”**。
2. 查看或修改各项配置。
3. 若需更改，点击右上角 **“编辑”** 按钮。
4. 修改完成后点击 **“保存”**。

> 📌 注意：**“通道单笔最大” > “用户单笔最大”**，即系统上限必须大于用户上限。

## **四、最佳实践建议**

| 场景                   | 建议                                                |
| ---------------------- | --------------------------------------------------- |
| **新增数字货币** | 优先使用主流稳定币（如 USDT, USDC），避免高波动资产 |
| **汇率设置**     | 入金汇率略高于出金汇率，形成合理利润空间            |
| **支付通道命名** | 使用清晰易懂的名称（如 “Bank Transfer - USD”）    |
| **手续费策略**   | 对小额交易免手续费，鼓励用户频繁操作                |
| **权限控制**     | 仅允许财务团队访问此模块，防止误操作                |

## **五、常见问题解答（FAQ）**

### **Q1：如何添加新的支付方式？**

- 在“支付通道”页面点击 **“新增”**，填写相关信息并绑定对应钱包即可。

### **Q2：为什么无法选择某个货币？**

- 请确认该货币已在“货币”模块中创建，并且类型正确（如数字货市 vs 法币）。

### **Q3：能否同时支持多个支付平台？**

- ✅ 可以。一个支付通道可关联多个平台（如 BO + MT5），但需确保接口兼容。

### **Q4：支付通道是否支持自动充值？**

- ❌ 当前版本不支持。需通过人工审核或定时任务触发。
