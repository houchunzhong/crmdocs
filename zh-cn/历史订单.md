# **历史订单**  

## **一、模块概览**

“历史订单”模块是 MIM CRM 的**交易行为分析中枢**，提供对 MT5、AQX 和 BO 三大交易系统中所有已结算或未平仓订单的全面视图。它支持按平台、账户、时间、类型等多维度筛选，帮助运营与合规人员：

- ✅ 回溯用户的完整交易路径  
- ✅ 分析盈利/亏损趋势与交易周期  
- ✅ 识别异常交易行为（如频繁短线、大额波动）  
- ✅ 支持高阶数据导出与报表生成  

> 🔐 仅限**管理员**、**运营主管**或**合规专员**访问。所有数据实时同步自交易服务器，确保准确性和时效性。

## **三、核心功能详解**

### **1. 平台切换功能**

#### **功能描述**  
顶部提供三个平台选项卡：**MT5**、**AQX**、**BO**，允许用户按交易系统分类查看订单数据。

#### **平台说明**
| 平台 | 说明 |
|------|------|
| **MT5** | MetaTrader 5 终端，适用于主流外汇、贵金属和差价合约交易 |
| **AQX** | TradeWise 自研量化交易系统，支持算法策略与API接入 |
| **BO** | Back Office 交易系统，常用于机构客户或内部测试账户 |

> 💡 示例：若需分析某客户的 MT5 账户交易习惯，可在 MT5 标签下进行筛选与统计。

---

### **2. 订单主视图（MT5 / AQX / BO）**

#### **功能描述**  
以表格形式展示指定平台下的全部历史订单，支持排序、搜索与分页浏览。

#### **字段说明**

##### **MT5 & AQX 视图**
| 字段 | 说明 |
|------|------|
| **订单号** | 唯一订单编号（如 `9988`），点击可查看详情 |
| **交易账号** | 关联的交易账户ID（如 `500067`），带“LIVE”或“DEMO”标识 |
| **交易品种** | 交易标的（如 `GBPUSD`, `EURUSD`, `XAUUSD`） |
| **类型** | 交易方向：SELL / BUY / ENTRY_IN / ENTRY_OUT |
| **交易量** | 开仓手数（如 `0.2`） |
| **平仓量** | 已平仓部分（如 `0.00` 表示未平仓） |
| **价位** | 成交价格（如 `1.36108`） |
| **时间** | 交易执行时间（精确到秒） |
| **佣金结算** | 该笔交易产生的佣金金额（通常为 `0.00`，除非有返佣） |
| **库存费** | 持仓过夜费用 |
| **交易盈亏** | 单笔交易浮动盈亏（绿色表示盈利，红色表示亏损） |

> ⚠️ 注意：MT5 和 AQX 数据不包含“利润百分比”字段，但可通过“交易盈亏”与“交易量”计算得出。

---

##### **BO 视图**
| 字段 | 说明 |
|------|------|
| **订单号** | 系统生成的唯一编号（如 `1_BTCUSD_...`） |
| **交易账号** | 账户ID（如 `1` 或 `501085`） |
| **状态** | 当前订单状态：  
  - 🟩 **盈利**：已获利  
  - 🔴 **亏损**：已亏损  
  - 🟨 **待处理**：尚未结算或正在处理中 |
| **交易品种** | 如 `BTCUSD`, `XAUUSD`, `AUDUSD` |
| **类型** | 上涨 / 下跌（即 Buy / Sell） |
| **交易量** | 手数（如 `10.00`） |
| **利润百分比** | 盈利比例（如 `+88%`） |
| **交易盈亏** | 实际金额盈亏（如 `+8.80`） |
| **交易周期** | 持仓时长（如 `30 s`） |
| **下单时间** | 提交订单的时间 |
| **开仓时间（本地）** | 实际开仓时间（本地时区） |
| **开仓价格** | 入场价格（如 `67092.19`） |

> 💡 示例：订单 `1_XAUUSD_...` 显示“亏损 -10.00”，表明该用户在黄金交易中损失了 $10。

### **3. 高级筛选与搜索功能**

#### **功能描述**  
提供多种条件组合查询，帮助用户快速定位目标订单。

#### **筛选器说明**
| 控件 | 功能 |
|------|------|
| **交易服务器** | 选择特定的交易服务器（如 `Options`, `Forex`, `Crypto`） |
| **组** | 按账户分组过滤（如“VIP客户”、“新注册用户”） |
| **LIVE / DEMO** | 区分实盘与模拟交易（关键区分项） |
| **订单号/交易账号搜索** | 输入关键词查找具体订单或账户 |

> 💡 建议：结合“LIVE”筛选 + 时间范围，专注于实盘交易行为分析。

## **四、数据分析应用指南**

### **1. 客户交易行为分析**

#### **场景：识别活跃客户**
- ✅ 方法：  
  1. 进入“历史订单” → 选择 MT5 → 设置时间范围（如过去7天）  
  2. 筛选“LIVE”订单  
  3. 按“交易账号”聚合统计订单数量  
  4. 排序后找出交易频率最高的用户

> 📊 输出：可生成“高活跃用户名单”，用于优先服务或奖励激励。

---

#### **场景：评估交易策略有效性**
- ✅ 方法：  
  1. 选取某一用户的所有订单  
  2. 按“交易品种”分组  
  3. 计算各品种平均盈亏与胜率  
  4. 对比发现其偏好于短周期高波动资产（如 BTCUSD）

> 📊 输出：判断用户是否适合推荐某些产品或提供风险管理建议。

---

### **2. 风险监控与合规审查**

#### **场景：识别异常交易模式**
- ✅ 指标：  
  - 频繁小额交易（如每分钟多次开仓）  
  - 极短交易周期（如 <10秒）  
  - 大额亏损后立即反向操作（可能为情绪化交易）  

> 🔍 操作：  
  1. 使用“交易周期”筛选 `< 30s` 的订单  
  2. 查看“交易盈亏”是否持续为负  
  3. 若符合特征，标记并通知合规部门

---

#### **场景：验证 KYC 风控逻辑**
- ✅ 方法：  
  1. 查询某高风险用户的历史订单  
  2. 检查其是否频繁交易高杠杆品种（如加密货币）  
  3. 结合 KYC 状态判断是否存在“身份不符”或“过度投机”风险

---

## **五、常见问题解答（FAQ）**

### **Q1：为什么 AQX 平台没有数据？**  
- ❌ 可能原因：  
  - 用户未在 AQX 平台交易  
  - 该用户仅使用 MT5 或 BO  
  - 数据同步延迟  

> ✅ 解决方案：确认用户是否绑定了 AQX 账户；等待 5–10 分钟刷新。

---

### **Q2：如何查看某个订单的详细信息？**  
- ✅ 方法：  
  1. 在“历史订单”列表中找到目标订单  
  2. 点击“订单号”列即可弹出详情面板  
  3. 查看完整的成交价、止损止盈、挂单类型等信息

---

### **Q3：“LIVE”和“DEMO”有什么区别？**  
- ✅ **LIVE**：实盘交易，涉及真实资金，影响盈亏与佣金  
- ✅ **DEMO**：模拟交易，无资金流动，仅用于学习或测试策略  

> 💡 建议：合规审查应重点关注 LIVE 订单。

---

### **Q4：能否查看未平仓订单？**  
- ✅ 是的。进入“BO持仓订单”子模块即可查看当前持有中的 BO 订单。

## **六、总结**

“历史订单”模块是连接交易行为与业务决策的关键桥梁。通过提供跨平台、多维度的数据视图，它不仅支持日常运营监控，更赋能深度数据分析与风险预警。无论是识别优质客户、优化服务策略，还是加强合规管理，本模块都提供了坚实的数据基础。

✅ **推荐操作流程**：  
1. 进入“历史订单” → 2. 选择平台（如 MT5）→ 3. 设置“LIVE”与时间范围 → 4. 筛选目标用户 → 5. 分析盈亏趋势与交易频率 → 6. 导出或上报结果

> 📌 所有数据均来自实时交易系统，真实反映用户行为，是客户画像、绩效评估与监管报告的重要依据。
