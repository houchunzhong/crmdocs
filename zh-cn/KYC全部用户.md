# **全部用户**  

## **一、模块概览**

“全部用户”模块是 MIM CRM 的**用户全景仪表盘**，集中展示平台内所有用户（包括已验证、待审核、被拒绝、资料不全等状态）的基本信息。它不仅提供强大的筛选与搜索能力，还支持一键跳转至用户详情页，便于深入审查KYC资料、交易行为或资金流动。

### ✅ 核心价值：
- 实现全量用户数据可视化
- 支持精细化分类与批量导出
- 提供历史审核记录与资料变更追踪
- 保障平台对用户生命周期的全面掌控

> 🔐 仅限 **系统管理员**、**合规官** 和 **财务主管** 访问。普通客户与 IB 无法查看此页面。

---

## **二、界面布局说明**

| 项目 | 功能 |
|------|------|
| **用户** | 进入用户管理主菜单 |
| - KYC 审核 | 展开下级菜单 |
|   - 待审核用户 (0) | 查看待审用户 |

|   - 待审核提款账户 (0) | 查看待审出金请求 |

---

### **全部用户列表视图**
| 字段 | 说明 |
|------|------|
| **用户** | 用户姓名或昵称，点击可进入详情页 |
| **KYC状态** | 显示当前身份验证状态（如“已验证”、“待审核”、“被拒绝”、“资料不全”） |
| **国籍** | 用户国籍（如 🇺🇸 United States） |
| **居住国家** | 用户常住国家 |
| **手机/邮箱** | 联系方式，支持复制 |
| **用户等级** | 用户类型（如 Verified Client, New Client） |
| **资料来源** | 注册渠道（如 Web、App、IB 邀请码） |
| **注册时间** | 用户首次注册时间 |
| **提交时间** | 用户上传资料的时间 |
| **上次审核时间** | 最近一次KYC审核的时间 |
| **操作** | 包含 **查看** 按钮，用于进入用户详情页进行深入分析 |

> 💡 示例：显示用户 `Thi Hà Vũ`，KYC状态为“已验证”，国籍越南，注册时间为 `09/02/2026`。

## **三、核心功能详解**

### **1. 多维度筛选与智能搜索**

#### **可用筛选项**
| 条件 | 说明 |
|------|------|
| **KYC状态** | 下拉选择：资料不全、待审核、已验证、被拒绝 |
| **国籍** | 可选国家列表，支持模糊匹配 |
| **注册时间从 ~ 到** | 按注册时间段过滤 |
| **用户ID/姓名/手机号/邮箱 查询** | 快速搜索特定用户 |

> ✅ 实用技巧：
> - 使用“KYC状态 = 资料不全”快速定位未完成身份信息填写的用户。
> - 组合“国籍 = Vietnam” + “KYC状态 = 已验证”可分析某国合规用户比例。

---

### **2. 一键跳转用户详情页**

#### **操作流程**
1. 在“全部用户”列表中找到目标用户（如 `Thi Hà Vũ`）
2. 点击 **查看** 按钮
3. 直接跳转至该用户的 **用户详情页**
4. 在“最新资料”标签页中查看其完整的身份信息、住址证明、银行信息等
5. 在“审核记录”标签页中查看历史审核结果与备注

> ✅ 优势：
> - 减少页面切换，提升工作效率
> - 所有操作均记录审计日志

---

### **3. 数据导出与报表生成**

#### **功能说明**
- 支持将当前筛选结果导出为 Excel 文件
- 导出内容包括：用户名、KYC状态、国籍、居住地、联系方式、注册时间、提交时间、审核时间等
- 不包含敏感材料（如身份证照片）

> ⚠️ 注意事项：
> - 仅限有 `user:export:excel` 权限的角色操作
> - 建议定期导出作为合规审计证据

---

### **4. 用户状态可视化与趋势分析**

#### **功能亮点**
- **颜色编码标识状态**：
  - ✅ 绿色：“已验证”
  - ⚠️ 黄色：“资料不全”
  - ❌ 红色：“被拒绝”
  - 🔍 灰色：“待审核”
- **实时统计总数**：顶部显示各状态用户数量（如“已验证：120”）
- **支持分页加载**：每页最多显示10条，支持翻页浏览

> ✅ 成果：直观反映平台整体合规水平与用户健康度。

## **四、典型使用场景**

### **场景 1：合规官执行季度合规审计**

> 🎯 目标：确保平台符合反洗钱（AML）与KYC法规要求

- ✅ 操作流程：
  1. 登录系统 → 进入 **用户 → KYC 审核 → 全部用户**
  2. 设置筛选条件：
     - KYC状态：`被拒绝`
     - 国籍：Singapore
  3. 查看当前被拒用户名单（如 3人）
  4. 点击 **查看** 进入每位用户详情页
  5. 检查拒绝原因是否合理（如“虚假信息”、“证件过期”）
  6. 若无异常 → 归档为合规报告附件
  7. 若发现误判 → 重新审核并更正状态

> ✅ 成果：满足监管机构的审计要求，降低法律风险。

---

### **场景 2：财务主管分析高净值客户分布**

> 🎯 目标：识别潜在大客户群体，制定营销策略

- ✅ 操作流程：
  1. 进入 **用户 → KYC 审核 → 全部用户**
  2. 设置筛选条件：
     - KYC状态：`已验证`
     - 注册时间：`01/01/2026 ~ 31/01/2026`
     - 国籍：Hong Kong, Singapore, Malaysia
  3. 查看符合条件的用户（如 87人）
  4. 点击 **下载到Excel** 导出名单
  5. 分析其入金金额、交易频率等数据
  6. 发现其中 12 名用户累计入金超 $10,000 → 标记为“高净值客户”

> ✅ 成果：为精准营销提供数据支持。

---

### **场景 3：系统管理员排查异常账户**

> 🎯 目标：发现可能的欺诈行为或技术问题

- ✅ 操作流程：
  1. 发现某用户频繁修改资料 → 进入 **全部用户**
  2. 搜索用户ID `16684`
  3. 查看其“审核记录”：
     - 第一次审核：通过
     - 第二次审核：拒绝，备注“False information”
     - 第三次提交：再次拒绝
  4. 判断为恶意尝试 → 点击 **查看** → 进入详情页
  5. 发现其邮箱与多个不同国家用户重复 → 标记为“高风险用户”
  6. 启动反欺诈流程：冻结账户、通知法务

> ✅ 成果：及时拦截潜在风险，保护平台安全。

## **六、常见问题解答（FAQ）**

### **Q1：为什么某个用户没有出现在“全部用户”列表中？**  
- ❌ 可能原因：用户已被注销、账号封禁或数据同步延迟  
- ✅ 解决方案：检查用户状态，或联系技术支持确认

---

### **Q2：能否批量审核多个用户？**  
- ❌ 当前版本不支持批量操作，需逐个审核  
- ✅ 未来版本将引入“批量通过/拒绝”功能

---

### **Q3：如何查看用户的历史审核记录？**  
- ✅ 进入用户详情页 → 切换至 **审核记录** 标签页即可查看所有历史审核结果与备注

---

### **Q4：KYC状态“资料不全”是什么意思？**  
- ✅ 表示用户已注册但未提交完整身份信息（如缺少出生日期、地址等），需补充后才能完成审核。

## **七、总结**

“全部用户”模块是 MIM CRM 实现**全局用户治理与合规监控**的关键组件。它不仅提供了清晰的用户全景视图，还支持高效的筛选、分析与导出功能，帮助团队在满足监管要求的同时，优化客户服务与风险管理。

✅ **推荐工作流**：  
1. 进入 **用户 → KYC 审核 → 全部用户** → 2. 筛选目标用户 → 3. 点击 **查看** → 4. 分析资料与审核记录 → 5. 执行必要操作

> 📌 所有操作均受权限控制，保障数据安全与合规性。
