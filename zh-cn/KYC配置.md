# **KYC 审核模块功能配置与使用说明书**  
**My Investment Markets (MIM) CRM 系统**

本手册详细说明 **MIM CRM** 中“KYC 审核”模块的功能架构、操作流程及配置方法，适用于管理员、合规专员和财务人员，用于高效管理用户身份验证、提款账户审核等关键合规流程。

---

## **一、模块概览**

“KYC 审核”模块是 MIM CRM 的核心合规组件，确保所有用户在进行交易或提款前完成身份识别与信息验证。该模块支持以下三大功能：

- ✅ **待审核用户（Pending Users）**：处理新注册用户的 KYC 提交请求
- ✅ **待审核提款账户（Pending Withdrawal Accounts）**：审核用户绑定的提款账户信息
- ✅ **全部用户 / 全部提款账户**：查看所有已通过或未通过审核的记录

> 🔐 所有审核操作均需由具备权限的管理员执行，并保留完整的操作日志以满足监管要求。

## **三、核心功能详解**

### **1. 待审核用户（Pending Users）**

#### **功能描述**
展示所有已提交 KYC 资料但尚未完成人工审核的用户。系统自动收集用户上传的身份证明文件（如护照、身份证、驾照）、地址证明（如水电账单）及联系方式。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **用户** | 用户名 + 用户ID（如 `zhizhizhi 17974`） |
| **KYC状态** | 当前审核状态：<br>🟢 `已验证`<br>🟡 `待审核`<br>🔴 `被拒绝` |
| **国籍** | 用户国籍（附国旗图标） |
| **居住国家** | 实际居住地（可能与国籍不同） |
| **手机/邮箱** | 联系方式，带验证状态标识：<br>✅ 已验证<br>⚠️ 未验证 |
| **用户等级** | 如 `Client`, `Verified Client`, `IB` |
| **资料来源** | 提交方式：`用户提交` 或 `系统导入` |
| **注册时间** | 用户注册时间戳 |
| **提交时间** | KYC 资料提交时间 |
| **上次审核时间** | 最近一次审核操作的时间 |
| **操作** | 按钮：**审核** 或 **查看** |

#### **操作流程**
1. 点击 **“审核”** 进入详情页。
2. 查看用户上传的证件照片（如护照正反面、住址证明）。
3. 核对信息真实性：
   - 是否为本人？
   - 是否有效期内？
   - 是否清晰可辨？
4. 选择结果：
   - ✅ **通过** → 用户升级为 `Verified Client`
   - ❌ **拒绝** → 填写原因并通知用户重新提交
5. 点击保存后，系统更新状态并发送通知。

> ⚠️ 注意：拒绝时必须填写明确理由，避免法律纠纷。

---

### **2. 待审核提款账户（Pending Withdrawal Accounts）**

#### **功能描述**
用于审核用户绑定的提款账户信息，包括银行卡、数字钱包（如 USDT 地址）、支付宝、微信等。确保收款人姓名、账号、币种、国家等信息准确无误，防止洗钱风险。

#### **字段说明**
| 字段 | 说明 |
|------|------|
| **用户ID** | 用户唯一标识（如 `17753`） |
| **审核状态** | 当前状态：<br>🟡 `待审核`<br>🟢 `审核通过`<br>🔴 `被拒绝` |
| **收款人姓名** | 银行卡持有人或钱包账户名 |
| **收款币种** | 如 USD, EUR, GBP, USDT, USDC |
| **类型** | 收款方式：<br>银行转账<br>数字钱包<br>第三方支付 |
| **收款账号/数字钱包地址** | 实际收款信息（如银行卡号、钱包地址） |
| **国家** | 收款账户所在国家 |
| **提交时间** | 用户提交时间 |
| **操作** | 按钮：**去审核** 或 **查看** |

#### **操作流程**
1. 点击 **“去审核”** 进入详情页。
2. 核查信息一致性：
   - 收款人姓名是否与用户一致？
   - 银行卡号格式是否正确？
   - 数字钱包地址是否有效？
3. 选择结果：
   - ✅ **通过** → 可用于后续提款
   - ❌ **拒绝** → 填写原因并提示用户修改
4. 保存后，系统同步状态至用户账户。

> 💡 建议：对高金额或异常地区账户（如虚拟货币、离岸账户）加强人工复核。

### **3. 全部用户 / 全部提款账户**

#### **功能描述**
提供完整的用户与提款账户历史记录，便于审计、报表生成和合规检查。

#### **用途**
- 日常对账与数据导出
- 监管机构抽查准备
- 内部绩效分析（如 IB 推荐客户 KYC 通过率）

#### **支持操作**
- 导出 Excel
- 按状态、国籍、时间等维度筛选
- 查看历史审核记录

## **四、审核状态说明**

| 状态 | 含义 | 对用户的影响 |
|------|------|----------------|
| **待审核** | 资料已提交，等待管理员处理 | 无法交易或提款 |
| **已验证** | 审核通过，信息真实有效 | 可正常交易、提款、提升信用等级 |
| **被拒绝** | 资料不合规或虚假 | 需重新提交；多次失败可能冻结账户 |

## **五、最佳实践建议**

| 项目 | 建议 |
|------|------|
| **权限管理** | 仅授予合规团队或指定管理员审核权限 |
| **双人复核机制** | 对大额账户或敏感国家用户启用双重审核 |
| **定期清理** | 定期检查长期未审核的申请，避免积压 |
| **自动化提醒** | 设置邮件/站内信提醒，及时处理待办任务 |
| **文档归档** | 所有审核记录应保留至少 5 年，符合 AML 要求 |

## **六、常见问题解答（FAQ）**

### **Q1：用户提交了 KYC 但显示“待审核”，多久能处理？**
- 建议在 **24 小时内** 完成审核，提升用户体验与信任度。

### **Q2：用户上传的是复印件，可以接受吗？**
- ❌ 不推荐。必须为原件扫描件或高清照片，确保防伪特征可见。

### **Q3：用户更换了提款账户，需要重新审核吗？**
- ✅ 是的。每次变更提款账户都需重新提交并经过审核。

### **Q4：如何批量处理多个用户？**
- 使用“全部用户”页面进行筛选后导出 Excel，再手动处理或提交给上级审批。
